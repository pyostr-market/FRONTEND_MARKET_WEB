# Каталог товаров с Infinite Scroll и сохранением позиции

## Обзор

Реализован компонент каталога товаров для маркетплейса со следующими возможностями:

- ✅ Подгрузка товаров порциями при скролле (lazy loading / infinite scroll)
- ✅ Сохранение состояния прокрутки при переходе на страницу товара и возврате
- ✅ Поддержка фильтров и сортировки
- ✅ Виртуализация списка для оптимизации производительности
- ✅ Кэширование данных в localStorage

## Функциональность

### 1. Подгрузка товаров (Infinite Scroll)

**API:** `/api/products`

**Параметры запроса:**
- `category_id` - ID категории
- `product_type_id` - ID типа товара
- `sort_type` - тип сортировки (`default`, `price_asc`, `price_desc`)
- `limit` - количество товаров (по умолчанию 12)
- `offset` - смещение для пагинации
- `attributes` - JSON с фильтрами

**Логика работы:**
1. При загрузке страницы загружается первая порция товаров (limit = 12)
2. При достижении конца списка автоматически загружается следующая порция
3. Для обычного ProductGrid используется Intersection Observer
4. Для ProductGridVirtual используется `onItemsRendered` из react-window

### 2. Сохранение позиции прокрутки

**Хук:** `useCatalogScroll`

**Механизм:**
1. При размонтировании компонента (уход на ProductPage) сохраняется в `sessionStorage`:
   - `scrollPos` - текущая позиция скролла
   - `productsCount` - количество загруженных товаров
   - `categoryId` - ID категории
   - `productType` - ID типа товара
   - `timestamp` - время сохранения

2. При возврате на страницу каталога:
   - Проверяется `sessionStorage` на наличие сохранённых данных
   - Проверяется, что категория не изменилась
   - Проверяется, что все товары отрендерены
   - Проверяется, что все изображения загружены
   - После успешного восстановления `sessionStorage` очищается
   - Устанавливается флаг `didRestoreScroll`

3. Проверка выполняется с интервалом 50мс до успешного восстановления

### 3. Фильтры и сортировка

**Хук:** `useFilterUrl`

**Синхронизация с URL:**
- Фильтры сохраняются в URL как `filter_{name}=["value1","value2"]`
- Сортировка сохраняется как `sort_type={value}`
- При изменении фильтров URL обновляется через `setSearchParams({ replace: true })`

**Применение фильтров:**
1. Пользователь выбирает значения фильтров
2. При нажатии "Применить" фильтры сохраняются в состоянии
3. URL обновляется
4. Массив товаров сбрасывается, offset = 0
5. Загружаются товары с новыми фильтрами

**Сброс фильтров:**
- Очищается массив товаров и состояние фильтров
- URL очищается от параметров фильтров
- Загружаются товары с дефолтными параметрами

### 4. Виртуализация списка

**Порог переключения:** 50 товаров

**Компонент:** `ProductGridVirtual` (react-window)

**Особенности:**
- В DOM рендерятся только видимые элементы + буфер (overscanRowCount = 2)
- Автоматическое вычисление количества колонок на основе ширины контейнера
- Фиксированная высота строки: 380px
- Подгрузка следующей порции при достижении конца списка

**Производительность:**
- Поддерживает 5000+ товаров без перегрузки браузера
- Минимальное потребление памяти
- Плавный скролл даже при большом количестве товаров

### 5. Кэширование

**Хук:** `useCatalog`

**Ключ кэша:**
```
catalogCache_v1_{category_id}_{product_type_id}_f_{JSON.stringify(filters)}
```

**Логика:**
1. При загрузке категории проверяется localStorage
2. Если кэш актуален (< 5 минут), данные восстанавливаются
3. При изменении состояния кэш обновляется
4. При смене категории кэш сбрасывается

**enableCache:** включено по умолчанию

## API

### GET /product/catalog/filters

Получение доступных фильтров для категории/типа товара

**Параметры:**
- `category_id` - ID категории
- `product_type_id` - ID типа товара
- `device_type_id` - альтернативное имя для product_type_id

**Ответ:**
```json
{
  "success": true,
  "data": {
    "filters": [
      {
        "name": "RAM",
        "is_filterable": true,
        "options": [
          {"value": "4 GB", "count": 5},
          {"value": "8 GB", "count": 10}
        ]
      }
    ]
  }
}
```

### GET /product

Получение списка товаров с фильтрацией

**Параметры:**
- `category_id` - ID категории
- `product_type_id` - ID типа товара
- `name` - поиск по названию
- `attributes` - JSON с атрибутами `{"RAM":["8 GB","16 GB"]}`
- `sort_type` - сортировка
- `limit` - количество (макс. 100)
- `offset` - смещение

**Ответ:**
```json
{
  "success": true,
  "data": {
    "total": 15,
    "items": [...]
  }
}
```

## Использование

### Базовое

```jsx
import CatalogPage from './pages/CatalogPage';

// Страница автоматически обрабатывает:
// - Загрузку товаров
// - Фильтры
// - Сортировку
// - Infinite scroll
// - Сохранение позиции
```

### Навигация

```jsx
import { Link } from 'react-router-dom';

// Переход в категорию
<Link to="/catalog?category=101">Смартфоны</Link>

// Переход с фильтром
<Link to="/catalog?category=101&filter_RAM=%5B%228%20GB%22%5D">
  С 8 ГБ RAM
</Link>

// Переход с сортировкой
<Link to="/catalog?category=101&sort_type=price_asc">
  По возрастанию цены
</Link>
```

- Массив товаров полностью восстановлен
- Изображения загружены

✅ **Производительность:**
- Каталог из 5000+ товаров не перегружает браузер
- FPS при скролле > 50
- Время восстановления позиции < 500мс

✅ **Интерактивность:**
- Фильтры работают без перезагрузки страницы

## Тестирование

1. Откройте каталог товаров
2. Проскролльте вниз для загрузки нескольких порций
3. Кликните на любой товар
4. Нажмите "Назад" в браузере
5. Убедитесь, что:
   - Позиция скролла восстановлена
   - Все товары отображаются
   - Изображения загружены

## Примечания

- Виртуализация включается автоматически при 50+ товарах
- Кэш хранится 5 минут
- sessionStorage очищается после успешного восстановления
- Intersection Observer настроен с rootMargin: 200px для плавной подгрузки
