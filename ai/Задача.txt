ЗАДАЧА ДЛЯ ИИ-АГЕНТА

Необходимо создать начальный фронтенд-каркас Marketplace на React.

Интерфейс навигации и header должен быть визуально и логически похож на интерфейс маркетплейса Ozon.

Проект должен строго соблюдать архитектуру, описанную в файле architecture.md.

Цель текущего этапа — создать базовую архитектуру приложения и базовый интерфейс с рабочей навигацией.

На данном этапе НЕ требуется сложная бизнес-логика.

---

ТЕХНОЛОГИИ

React
React Router
React Icons

Проект использует **JavaScript и JSX**.

TypeScript использовать **запрещено**.

Форматы файлов:

.js — логика, API, конфигурации
.jsx — React компоненты

Все React компоненты должны использовать **функциональные компоненты**.

Class components использовать нельзя.

---

ГЛАВНЫЕ ПРАВИЛА АРХИТЕКТУРЫ

Необходимо строго соблюдать архитектурные слои:

app
pages
widgets
features
entities
shared

Запрещено:

• писать API вызовы внутри UI компонентов
• писать бизнес-логику внутри компонентов
• использовать строковые URL
• смешивать слои архитектуры

UI компоненты должны:

• получать данные через props
• не содержать бизнес-логики
• не выполнять API запросы

---

СТРУКТУРА ПРОЕКТА

Создать структуру:

src

app
app/router
app/providers

pages
pages/HomePage
pages/CatalogPage
pages/CartPage
pages/WishlistPage

widgets
widgets/Header
widgets/MobileHeader
widgets/MobileNavbar
widgets/SearchOverlay

features
features/search

shared
shared/ui
shared/api
shared/hooks
shared/constants

---

ГЛАВНАЯ ТОЧКА ПРИЛОЖЕНИЯ

Создать файл:

App.jsx

Структура приложения должна быть:

BrowserRouter
AppProviders
AppRouter

AppProviders пока может просто возвращать children.

---

РОУТИНГ

Создать систему маршрутов.

Файл:

app/router/paths.js

Там должны быть маршруты:

/
/catalog
/cart
/wishlist

Файл:

app/router/routes.js

Там должна находиться конфигурация страниц.

Каждый маршрут должен содержать:

path
component
layout
public/private

Файл роутера:

app/router/AppRouter.jsx

Использовать React Router.

---

СТРАНИЦЫ

Создать страницы:

HomePage
CatalogPage
CartPage
WishlistPage

Каждая страница должна находиться в отдельной папке.

Каждая страница пока отображает простой текст:

Home Page
Catalog Page
Cart Page
Wishlist Page

---

DESKTOP HEADER (как у Ozon)

Создать widget Header.

Header должен отображаться на всех страницах.

Header должен содержать:

логотип (Marketplace)
кнопку "Каталог"
поле поиска
иконки действий

Иконки:

Профиль
Избранное
Корзина

Использовать React Icons:

FiUser
FiHeart
FiShoppingCart

Поисковая строка должна занимать основную ширину header, как у Ozon.

Кнопка "Каталог" пока не должна открывать меню.

Header должен отображаться **только на desktop**.

---

MOBILE HEADER

Создать MobileHeader.

Он должен отображаться только на мобильных устройствах.

MobileHeader должен содержать:

логотип
поиск
иконку корзины

Высота header должна быть компактной.

---

MOBILE NAVIGATION (как у Ozon)

Создать MobileNavbar.

Это нижняя навигационная панель.

Она должна:

быть фиксированной
располагаться внизу экрана
быть видимой только на мобильных устройствах

Использовать React Icons.

Кнопки:

Главная
Каталог
Корзина
Избранное
Профиль

Иконки:

FiHome
FiGrid
FiShoppingCart
FiHeart
FiUser

---

ПОВЕДЕНИЕ НАВИГАЦИИ

Главная → HomePage
Каталог → CatalogPage
Корзина → CartPage
Избранное → WishlistPage

---

ПРОФИЛЬ

Кнопка профиль должна открывать модальное окно.

Модальное окно должно содержать:

заголовок
поле ввода телефона

API пока не требуется.

---

SEARCH OVERLAY

Создать компонент:

SearchOverlay

Overlay должен открываться когда пользователь:

кликает на поиск
начинает вводить текст

Overlay должен перекрывать страницу.

Он должен содержать:

поле поиска
список популярных товаров

Популярные товары (заглушка):

iPhone 15
MacBook Pro
Apple Watch
AirPods Pro

---

ПОИСК

Создать feature:

features/search

Создать функцию:

searchProducts(query)

Файл:

shared/api/searchApi.js

Функция должна возвращать Promise.

Пример результата:

[
"iPhone 15",
"Samsung Galaxy S24",
"MacBook Pro",
"Apple Watch",
"AirPods Pro"
]

При вводе текста список должен фильтроваться.

Результаты отображаются в SearchOverlay.

---

LAYOUT

Создать основной layout приложения.

MainLayout должен:

показывать Header на desktop
показывать MobileHeader на mobile
показывать MobileNavbar на mobile
рендерить страницу

Все страницы должны отображаться внутри MainLayout.

---

АДАПТИВНОСТЬ

Desktop:

виден Header
скрыт MobileHeader
скрыт MobileNavbar

Mobile:

скрыт Header
виден MobileHeader
виден MobileNavbar

Использовать media queries.

---

ОЖИДАЕМЫЙ РЕЗУЛЬТАТ

После реализации:

приложение запускается
структура проекта соответствует architecture.md
работают маршруты
отображается desktop header
отображается mobile header
работает mobile navigation
открывается search overlay
работает мок-поиск

На этом этапе сложная бизнес-логика не требуется.
