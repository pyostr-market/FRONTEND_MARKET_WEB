architecture.md

Цель этого документа — описать архитектуру фронтенд‑приложения Marketplace на React. Документ предназначен для использования ИИ‑агентом и разработчиками как источник правил и принципов, которые необходимо соблюдать при разработке новых модулей, страниц и компонентов.

Проект реализуется на React и использует JavaScript (JS/JSX). TypeScript в проекте не используется.

Форматы файлов:
.js — логика, конфигурации, API, утилиты
.jsx — React компоненты

Использование .ts и .tsx запрещено.

---

ОБЩИЕ ПРИНЦИПЫ

Приложение является большим маркетплейсом с большим количеством данных и интеграций с backend сервисами.

Backend реализован на Python и предоставляет данные через HTTPS API.

Архитектура фронтенда должна обеспечивать:

масштабируемость
разделение ответственности
предсказуемую структуру проекта
удобство генерации кода ИИ‑агентами

Основные правила:

1. Все приложение разделено на архитектурные слои.
2. Каждый слой имеет четкую зону ответственности.
3. UI компоненты не содержат бизнес‑логики.
4. API вызовы выполняются только через API слой.
5. Навигация использует централизованную систему маршрутов.
6. Запрещено использовать строковые URL напрямую.
7. Повторно используемые элементы должны находиться в shared.
8. Состояние приложения разделяется на серверное и клиентское.
9. Компоненты должны быть максимально изолированы.
10. Архитектура должна позволять легко добавлять новые страницы и функции.

ФИЛЬТРЫ

Фильтры каталога должны храниться в URL.

КОМПОНЕНТЫ

UI компоненты:

получают данные через props
не содержат бизнес‑логики
не выполняют API запросы

---

HOOKS

Вся логика компонентов должна выноситься в hooks.

Примеры:

useProduct
useCatalog
useFilters
useSearch

---

ПРОИЗВОДИТЕЛЬНОСТЬ

Использовать lazy loading страниц.

Кэшировать API запросы.

ОБЩИЕ ПРАВИЛА

ЦЕЛЬ АРХИТЕКТУРЫ

Создать масштабируемую архитектуру маркетплейса, позволяющую:

легко добавлять функции

без необходимости переписывать существующий код.
