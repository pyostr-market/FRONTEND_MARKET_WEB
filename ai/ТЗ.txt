🧩 Техническое задание: Каталог товаров + фильтры + корзина
1. Общая логика страницы каталога
1.1 Точка входа

Пользователь попадает в каталог:

через выбор категории → category=<id>

через выбор типа устройства → productType=<id> (в API: device_type_id)

1.2 Начальная загрузка

При открытии страницы:

Отправляется запрос на:

получение списка товаров (без фильтров)

получение доступных фильтров

Параметры запроса:

category или device_type_id (если есть)

без дополнительных фильтров

2. Работа с API
2.1 Получение товаров

Endpoint: /products

Query params:

category

device_type_id

filters (набор выбранных фильтров)

page / limit

Поведение:

при первой загрузке → без фильтров

при нажатии "Показать" → с выбранными фильтрами

2.2 Получение фильтров

Endpoint: /filters

Query params:

category или device_type_id

Ответ:

список атрибутов:

name

values[]

3. Фильтры
3.1 Десктоп

Расположение: слева (sidebar)

Структура:

Название фильтра

Список значений (checkbox)

3.2 Логика работы

При клике на фильтр:

состояние сохраняется локально

запрос НЕ отправляется

показывается кнопка "Показать"

При нажатии "Показать":

отправляется запрос /products

каталог обновляется

3.3 Мобильная версия

Фильтры скрыты

Кнопка: "Фильтры"

По нажатию:

открывается модальное окно

внутри:

список фильтров

кнопка "Применить"

4. Каталог товаров
4.1 Сетка

Десктоп: ~3 товара в ряд

Адаптив:

минимум: 2

максимум: 4 (в зависимости от ширины)

4.2 Карточка товара

Содержимое:

📷 Изображение (квадрат)

карусель (свайп / точки)

без стрелок

индикаторы (dots)

📝 Название

💰 Цена

🛒 Добавление в корзину:

кнопка "Добавить"

после добавления:

счетчик [-] 1 [+]

максимум: 10 шт

при уменьшении до 0 → удаление

5. Пагинация / подгрузка
5.1 Формат

Infinite scroll (автоподгрузка)

5.2 Параметры

limit: 10–20 товаров

при скролле вниз:

подгружается следующая страница

добавляется к списку

6. Хедер и поведение при скролле
6.1 Sticky элементы

Фиксируются:

верхний хедер (поиск, логотип, корзина)

sidebar с фильтрами

Скрывается при скролле:

блок с типами устройств

7. Корзина
7.1 Общая логика

хранение количества товаров (по id)

максимум: 10 единиц на товар

7.2 Отображение
Десктоп:

иконка корзины в хедере

бейдж с количеством товаров (сумма всех позиций)

Мобильная версия:

иконка в нижнем navbar

тот же счетчик

👉 Счетчик должен быть единым состоянием (global state)

8. Сортировка

Добавить UI (без логики):

по цене

по популярности

9. Мобильная адаптация
9.1 Сетка

адаптивная (2–4 карточки)

9.2 Фильтры

кнопка → модалка

9.3 Навигация

нижний navbar:

корзина (со счетчиком)

10. Состояние (State Management)

Рекомендуется:

Global store (например: Redux / Zustand / Vuex)

Хранить:

выбранные фильтры (до применения)

примененные фильтры

корзину

список товаров (с пагинацией)

11. UX детали

Лоадеры:

при первой загрузке

при подгрузке товаров

Пустое состояние:

"Товары не найдены"

Debounce не нужен (нет авто-запросов)
